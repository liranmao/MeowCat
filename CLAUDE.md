## Project Overview

MeowCat (Multi-resolution Omics informed Whole-slide Cell Annotation Tool) is a deep learning framework for cell-type annotation in H&E histopathology images using spatially-registered transcriptomics data as supervision. It predicts cell-type distributions across whole-slide images at pixel resolution.

Two supervision types: **Visium** (spot-level soft proportions via RCTD deconvolution, MSE loss) and **Xenium/manual** (single-cell hard labels, cross-entropy loss).

## Build & Run

```bash
pip install -e .                    # editable install (only declares pyyaml; heavy deps via conda)
meowcat <step> --config config.yaml [--samples S1,S2] [--dry-run]
pytest tests/                       # pure-Python tests, no GPU/data needed
```

No linter is configured.

## Conda Environments

Three separate conda envs are required (the CLI spawns subprocesses into each):
- **RCTD** — R with spacexr, Seurat v5 (Step 1)
- **rapids_singlecell** — image preprocessing, feature extraction (Steps 2, 3)
- **he_anno** — batching, training, prediction, visualization (Steps 4–7)

## CLI & Pipeline Architecture

Entry point: `meowcat.cli:main` (defined in `pyproject.toml`). The CLI is a thin subprocess orchestrator — it reads a YAML config, builds shell commands via `meowcat/pipeline.py`, and runs them with `subprocess.run()`. No heavy ML imports happen in the CLI itself.

**Pipeline steps** (in order):
1. `rctd` — R script deconvolves Visium spots into cell-type proportions
2. `check-resolution` — audits image MPP vs target
3. `preprocess` — **data-agnostic** (works for both Visium and Xenium). 5 sub-steps per sample (pixel size, rescale/pad, tissue mask, UNI feature extraction, feature fusion → single_super_emb.h5ad). Supports `--start-from N` for resuming. For Xenium, `single_super_emb.h5ad` is automatically merged into the cellbin as `histology_2048` during `prepare-xenium-batches`.
3.5. `prepare-visium` — Visium metadata + embeddings per sample (anno-names.txt, anno_matrix.tsv, locs.tsv, radius.txt, embeddings-hist.pickle). Aligns RCTD barcodes with tissue positions to ensure row-level consistency between anno_matrix.tsv and locs.tsv.
4. `prepare-visium-batches` / `prepare-xenium-batches` — tokenize spots/cells, coreset subsample, write `.npy` batch files
5. `train` — PyTorch Lightning training (multi-phase: reconstruction pretrain → Visium MSE → Xenium CE)
6. `predict` + `visualize` — full-grid inference, argmax maps with custom color map (`config/visualization_cmap.json`), intensity maps
7. `slide` — assembles results into PowerPoint

`run-all` executes steps 1–7 in sequence (includes prepare-visium and prepare-xenium-batches).

**Xenium data layout**: All Xenium data lives under `project.data_root/SAMPLE/` (same root as Visium).

User-provided:
- `he_raw.<ext>` — raw H&E image (input to `meowcat preprocess`)
- `xenium_raw/` — 10x Xenium output (cell_feature_matrix.h5, cells.parquet)
- `adata_cellbin_HistoSweep.h5ad` — cell-bin location mapping (from external HistoSweep alignment; `.obsm['transformed_pxl_loc_in_morphology']` has bin pixel coordinates)
- `annotation.csv` — cell-type annotations (cell_id, cell_state columns)

Generated by pipeline:
- `single_super_emb.h5ad` — UNI histology features (Step 3: `meowcat preprocess`)
- `Xenium_adata_cellbin_analysis_qv20.h5ad` — cellbin with `histology_2048` merged (Step 4)
- `Xenium_adata_cell.h5ad` — cell data with mapped annotations (Step 4)

During `prepare-xenium-batches`, histology features from `single_super_emb.h5ad` are merged into the cellbin as `.obsm['histology_2048']`. The pipeline errors if `single_super_emb.h5ad` is missing — run `meowcat preprocess` first.

## Config System

All parameters live in a single YAML file. `meowcat/config.py` uses Python dataclasses (`MeowCatConfig` with nested section dataclasses) to provide typed defaults and load overrides. Unknown keys are silently ignored. See `config/default.yaml` for all options (9 sections: project, rctd, preprocess, batches, xenium, train, predict, visualize, slide).

## Model Architecture

- **UNI ViT-Large**: External foundation model for histology patch embeddings (not trained, weights loaded)
- **MultiTaskModel** (`Train_predict/train_by_batch_cdan5.py`): 4-layer MLP encoder (`net_lat`) → 256-dim latent, cell-type head, reconstruction head, domain head with Gradient Reversal Layer (CDAN)
- **MultiResolutionModel** (`Train_predict/train_by_batch_cdan5_trainc_final2.py`): Extends MultiTaskModel with resolution embedding to distinguish Visium (res=0) from Xenium (res=1) data
- **Ensemble**: Multiple independent replicas (`n_states`); at prediction, latent embeddings aggregated with median, probabilities with mean

## Key Conventions

- **Batch file naming**: `batch_vis_XXX_x/y/d.npy` (Visium) vs `batch_xen_XXX_x/y/d.npy` (Xenium). The `vis`/`xen` prefix determines loss function during training.
- **Multi-phase training**: Phase 0 (masked reconstruction MSE) → Phase 1 (Visium MSE on soft labels) → Phase 2 (Xenium CE on hard labels). Encoder freezing at phase transitions.
- **Import style**: Training/preprocessing scripts use `sys.path` manipulation for sibling imports rather than package-relative imports.
- **Shared utilities**: `Preprocess/utils.py` (I/O helpers), `Preprocess/image.py` (disk masks), `Preprocess/ExtractFeatures/UTILS.py` (low-level image I/O, CUDA monitoring).
